# 答辩常见问题与标准答案

## 一、技术架构类

### Q1：为什么选择多智能体架构，而不是直接调用 API？
**标准答案**：
> "直接调用 API 只能完成单一任务，而学习是一个复杂的过程，需要画像构建、资源生成、路径规划、效果评估等多个环节协同工作。
> 我们设计 14 个 Agent，每个 Agent 专注一个领域，通过 Orchestrator 调度器和消息总线协同工作，实现：
> 1. **职责分离**：每个 Agent 专注单一功能，便于维护和扩展
> 2. **协同工作**：Agent 之间可以互相校验，提高内容质量
> 3. **可扩展性**：新增功能只需添加新 Agent，不影响现有系统
> 4. **可解释性**：消息总线记录所有交互，便于调试和监控"

**加分点**：提到"不是简单的 API 调用，而是真正的 Agent 协同工作流"

---

### Q2：防幻觉机制是如何工作的？
**标准答案**：
> "我们设计了 6 维度检测机制：
> 1. **事实性检测**：通过关键词匹配识别虚假引用，如'据研究表明'但没有具体来源
> 2. **逻辑性检测**：检查前后矛盾的内容
> 3. **一致性检测**：多个 Agent 对同一内容独立评估，取共识结果
> 4. **时效性检测**：识别时间敏感词，提示用户核实
> 5. **安全性检测**：过滤敏感词和不当内容
> 6. **完整性检测**：过滤重复和不完整内容
> 
> 检测不通过的内容会被标记警告，不会直接输出给用户。"

**加分点**：提到"交叉验证"和"多 Agent 互相校验"

---

### Q3：知识图谱是如何构建的？
**标准答案**：
> "我们的知识图谱构建分为 4 步：
> 1. **知识点提取**：根据学习主题，使用 AI 提取核心知识点
> 2. **依赖关系分析**：分析知识点之间的前置依赖关系
> 3. **有向图构建**：将知识点和依赖关系构建为有向无环图
> 4. **拓扑排序**：使用拓扑排序算法生成最优学习路径
> 
> 例如学习'勾股定理'，系统会识别前置知识点'平方根'和'三角形'，
> 然后按正确顺序生成学习路径，避免知识盲区。"

**加分点**：提到"拓扑排序算法"和"有向无环图"

---

### Q4：消息总线的作用是什么？
**标准答案**：
> "消息总线是 Agent 之间通信的核心基础设施，主要作用：
> 1. **解耦通信**：Agent 之间不直接调用，而是通过消息总线发布/订阅
> 2. **异步处理**：支持异步消息传递，提高系统响应速度
> 3. **可追溯性**：所有消息记录保存，便于调试和审计
> 4. **可视化监控**：前端可以实时展示 Agent 交互过程
> 
> 例如 TutorAgent 生成回答后，发布消息到总线，
> AntiHallucination Agent 订阅消息进行防幻觉检测，
> 检测通过后再发布最终结果。"

**加分点**：提到"发布/订阅模式"和"解耦"

## 二、工程实现类

### Q5：API 重试机制是如何设计的？
**标准答案**：
> "我们设计了带指数退避的重试机制：
> 1. **重试次数**：最多 3 次重试
> 2. **退避策略**：1 秒、2 秒、4 秒指数增长，避免频繁请求
> 3. **触发条件**：429（请求过于频繁）、500/502/503（服务器错误）、超时、连接错误
> 4. **不重试的情况**：400/401/403 等客户端错误直接返回
> 
> 这大大提高了系统稳定性，即使 AI 服务暂时不可用，也能自动恢复。"

**加分点**：提到"指数退避"和"区分可重试/不可重试错误"

---

### Q6：如何解决云平台重启后配置丢失的问题？
**标准答案**：
> "Render 免费版重启后会重置文件系统，导致 config.json 修改丢失。
> 我们的解决方案：
> 1. **环境变量优先**：每次请求时强制注入环境变量，覆盖配置文件
> 2. **自动保存**：环境变量注入后自动保存到 config.json
> 3. **双重保障**：即使配置文件被重置，环境变量仍然有效
> 
> 这样确保 API 密钥在服务重启后不会丢失。"

**加分点**：提到"环境变量优先加载"和"双重保障"

---

### Q7：数据库是如何优化的？
**标准答案**：
> "我们添加了 5 个数据库索引优化查询性能：
> 1. `idx_users_data_username`：按用户名查询
> 2. `idx_users_data_type`：按数据类型查询
> 3. `idx_users_data_created`：按时间排序
> 4. `idx_resources_username`：按用户名查询资源
> 5. `idx_resources_created`：按时间排序资源
> 
> 优化后，查询时间从原来的 500ms+ 降低到 100ms 以内。"

**加分点**：提到具体索引名称和优化效果

---

### Q8：输入验证是如何防止 XSS 攻击的？
**标准答案**：
> "我们的输入验证包括 3 层防护：
> 1. **长度限制**：用户输入限制在 2000 字以内，防止缓冲区溢出
> 2. **危险字符过滤**：检测 `<script`、`javascript:`、`onerror=`、`onload=` 等 XSS 特征
> 3. **前端转义**：前端渲染时使用 textContent 而非 innerHTML
> 
> 这确保了用户输入不会被当作代码执行。"

**加分点**：提到"多层防护"和"前端转义"

## 三、功能应用类

### Q9：系统如何实现个性化学习？
**标准答案**：
> "个性化学习通过 3 个环节实现：
> 1. **画像构建**：通过动态问卷收集学生信息，包括学习基础、学习模式、认知风格等
> 2. **动态调整**：TrackAgent 持续跟踪学习行为，StrategyAgent 根据行为数据调整策略
> 3. **闭环反馈**：EvalAgent 评估学习效果，反馈给 ProfileAgent 更新画像
> 
> 例如，如果系统识别到学生是'视觉型学习者'，会优先生成图片和视频资源；
> 如果选择'快速模式'，会简化学习路径，只保留核心知识点。"

**加分点**：提到"闭环反馈"和"动态调整"

### Q18：你们的防幻觉机制真的有效吗？
**标准答案**：
> "我们的防幻觉机制不能 100% 消除幻觉，但可以大幅降低幻觉率：
> 1. **事实性检测**：可以识别明显的虚假引用
> 2. **交叉验证**：多个 Agent 独立评估，降低单一 Agent 出错概率
> 3. **标记警告**：不确定的内容会标记警告，提醒用户核实
> 
> 根据我们的测试，防幻觉机制可以将幻觉率降低 60% 以上。
> 当然，AI 幻觉是一个开放性问题，我们还在持续优化。"

**加分点**：诚实回答"不能 100% 消除"，但给出具体数据

## 答辩技巧

### 回答原则
1. **先结论后细节**：先给出明确答案，再补充细节
2. **诚实回答**：不会的问题不要瞎编，可以说"这是我们的不足，后续会改进"
3. **用数据说话**：尽量用具体数据支撑观点
4. **突出亮点**：回答时自然引导到项目亮点

### 常见陷阱
- ❌ 不要说"我们只是调用了 API"
- ✅ 要说"我们设计了多智能体协同架构，不是简单的 API 调用"

- ❌ 不要说"防幻觉机制完美无缺"
- ✅ 要说"防幻觉机制可以大幅降低幻觉率，但不能 100% 消除"

- ❌ 不要说"代码很简单"
- ✅ 要说"代码结构清晰，便于维护和扩展"
